# 奖金池优化快速开始指南

## 第一步：执行数据库脚本

在 Supabase SQL 编辑器中运行 `bonus-pool-optimization.sql` 文件。

这将创建 `bonus_pool_monthly` 缓存表。

## 第二步：初始化历史数据

### 最简单的方法（推荐）

1. **打开奖金池管理页面**
   - 登录系统
   - 点击侧边栏的"奖金池"菜单

2. **点击"初始化数据"按钮**
   - 在页面右上角，刷新按钮旁边
   - 点击后会弹出确认对话框

3. **确认操作**
   - 点击"确定"开始初始化
   - 系统会自动重新计算所有历史月份的奖金池数据
   - 完成后会显示累计奖金池总额

4. **完成！**
   - 初始化完成后，再次打开奖金池页面会明显变快

## 注意事项

- ⚠️ **首次使用必须初始化**，否则仍然会很慢
- ⏱️ 初始化可能需要一些时间，取决于历史数据量
- ✅ 初始化只需要执行一次
- 🔄 之后每次查看奖金池会自动使用缓存，速度很快

## 如果"初始化数据"按钮不可见

确保您是以**管理员**或**商人**身份登录的，员工端不显示此按钮。

## 验证优化效果

初始化完成后：
- 再次打开奖金池页面应该明显变快（从几秒变成不到1秒）
- 数据应该和之前一致

## 遇到问题？

如果初始化失败或数据不准确，可以：
1. 再次点击"初始化数据"按钮重新计算
2. 检查数据库连接是否正常
3. 查看浏览器控制台的错误信息
