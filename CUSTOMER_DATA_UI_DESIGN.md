### 界面元素说明

#### 1. 文本输入区域

**组件**：`TextArea`（Ant Design）

**属性**：
- `rows={10}` - 显示10行
- `placeholder="请在此粘贴手机号码（每行一个）"`
- `value={textContent}`
- `onChange={handleTextChange}`
- `disabled={uploading}` - 上传时禁用

**功能**：
- 支持多行文本输入
- 支持直接粘贴
- 自动去除前后空格（可选）

#### 2. 上传按钮

**功能**：
- 解析文本内容

**点击后流程**：

### 上传逻辑详细说明

#### 文本解析

```javascript
// 1. 按行分割
const lines = textContent.split('\n');

// 2. 去除空行和空格
const phones = lines
  .map(line => line.trim())
  .filter(line => line.length > 0);

// 3. 前端去重
const uniquePhones = [...new Set(phones)];

// 4. 验证格式（可选）
const validPhones = uniquePhones.filter(phone => {
  // 简单的手机号码格式验证（11位数字）
  return /^1[3-9]\d{9}$/.test(phone);
});
```

#### 批量上传

```javascript
// 使用 Promise.allSettled 处理批量插入
const results = await Promise.allSettled(
  validPhones.map(phone => 
    supabase.from('customer_pool').insert({
      customer_phone: phone,
      uploaded_by: currentUser.name,
      status: 'pending'
    })
  )
);

// 统计结果
const success = results.filter(r => r.status === 'fulfilled').length;
const duplicates = results.filter(r => 
  r.status === 'rejected' && r.reason?.code === '23505' // UNIQUE约束错误
).length;
const errors = results.filter(r => 
  r.status === 'rejected' && r.reason?.code !== '23505'
).length;
```

## 交互流程

### 商人端上传流程

```
1. 用户打开"客户数据"页面
   ↓
2. 在文本框中粘贴手机号码
   ↓
3. 点击"上传"按钮
   ↓
4. 系统解析文本
   ↓
5. 前端去重
   ↓
6. 批量插入数据库
   ↓
7. 数据库自动去重（UNIQUE约束）
   ↓
8. 显示上传结果
   ↓
9. 更新统计信息
```

### 员工端获取流程

```
1. 用户打开"客户数据"页面
   ↓
2. 查看待分配数量
   ↓
3. 点击"获取数据"按钮
   ↓
4. 系统调用分配函数
   ↓
5. 检查待分配数量
   ↓
6. 如果为0，自动重置并打乱
   ↓
7. 随机选择一条手机号码
   ↓
8. 弹窗显示手机号码
   ↓
9. 用户操作：
   - 点击"复制" → 复制到剪贴板
   - 点击"再发一条" → 获取下一条
```

## 错误处理

### 上传错误场景

1. **空内容**：提示"请输入手机号码"
2. **格式错误**：提示"部分手机号码格式不正确"
3. **网络错误**：提示"网络连接失败，请重试"
4. **数据库错误**：提示"上传失败，请检查数据"

### 获取错误场景

1. **无可用数据**（理论上不会发生，因为会自动重置）
2. **网络错误**：提示"获取失败，请重试"
3. **函数调用错误**：提示"系统错误，请联系管理员"

## 数据验证

### 手机号码格式验证（可选）

```javascript
// 中国手机号码格式：1开头，11位数字
const phoneRegex = /^1[3-9]\d{9}$/;

// 验证函数
function validatePhone(phone) {
  return phoneRegex.test(phone.trim());
}
```

**注意**：如果不需要严格验证，可以只检查非空即可。

## 性能考虑

1. **批量上传**：使用批量插入，避免逐条插入
2. **去重优化**：前端去重减少数据库压力
3. **统计缓存**：统计信息可以缓存，减少查询频率
4. **异步处理**：所有操作使用异步，不阻塞界面
